面向征信补全的婚姻状况标签挖掘与轻量化落地

先用两阶段树模型完成近亿级稳定打标，再用 Multi-Agent 补足难例、解释与冷启动，最后以结构编码、参数高效微调和蒸馏探索低成本规模化。

Situation｜字段缺失影响画像与风险识别

银行征信中的婚姻字段存在缺失，而预研显示离婚群体逾期 30+ Lift 约为 1.81。这意味着婚姻状态不仅是画像补全字段，也会影响白名单授信、风险分层和后续人群策略。

当时的约束并不只是“训练一个分类器”：近亿级批处理要求低成本和稳定性，新标签与少样本场景要求快速冷启动，业务评审还需要逐条解释判定依据。单一模型很难同时满足这三类目标。

Task｜搭建可规模化、可解释、可压缩的标签挖掘链路

我的目标是把婚姻状态标签挖掘拆成三层能力：第一层用 XGBoost / TabPFN 完成稳定的大规模打标；第二层用 Multi-Agent 补足难例、解释与冷启动；第三层用表格 LLM 微调和蒸馏探索低成本部署。

这个拆分的核心不是让 LLM 替代树模型，而是让不同模型承担各自最合适的部分：树模型负责近亿级稳态任务，Agent 负责解释和难例，蒸馏模型负责把高成本语义能力压到可部署范围。

Action｜三层方案：树模型打底，Agent 补难例，小模型承接规模化

第一步是规模化底座。 我采用两阶段级联建模，以 XGBoost 为主路径，并探索 TabPFN 在表格小样本任务上的补充价值。评估不仅看 OOT AUC / KS，也看高分段风险识别率，避免离线排序指标与业务使用脱节。

第二步是 Agent 增量层。 我将结构化行转换为键值对或 Markdown 文本，把标注样本写入向量库，通过 RAG + Few-shot 动态检索相似案例；新旧标签由意图路由进入不同路径；并行专家分别分析不同证据，最终用类别 token 的 logprob 作为可排序置信度。完整链路可以概括为：

特征提取 → SQL 生成 → 逻辑判断 → 报告生成

第三步是微调与蒸馏。 表格压成一维文本会损失列身份和二维位置，因此加入 coltag 列标记与 pos2d 单元格坐标。教师模型使用 Qwen3.5-9B，配合 LoRA / DoRA 做生成式 SFT；学生模型为 Qwen3.5-0.8B。蒸馏目标为：

$$ \mathcal{L}_{KD} =\alpha\,\mathrm{CE}(y,p_s) +(1-\alpha)T^2\cdot \mathrm{KL}\!\left(p_t^{(T)}\parallel p_s^{(T)}\right). $$

LoRA 的低秩更新可写为 $\Delta W=BA$；DoRA 将更新拆成幅度与方向；AdaLoRA 按重要性动态分配 rank。实验结论是：少样本下，结构编码和数据量通常比单纯更换 PEFT 变体更重要。

业务数据不便公开时，我用 heart、blood、diabetes、car、creditg 等 TabLLM 公开数据集复现同一套序列化、PEFT、结构编码和蒸馏管线。代表性结构编码结果如下：

Result｜近亿级补标、难例补回与技术沉淀

传统建模在 OOT 上取得 AUC 0.75、KS 0.37，补齐约 9,957 万用户婚姻标签；评分 Top 3% 区域的风险识别率达到大盘 1.51 倍。Agent 侧补回 XGBoost 漏召的约 10% 难识别样本，在 60% 召回率下查准率达到 43%，相对 XGBoost 提升约 20%。

在公开集上，32-shot 下 LoRA 与 DoRA 接近，AdaLoRA 多数更弱；4-shot 时 LoRA 相对 XGBoost 更占优，样本增加后两者互有胜负。公平蒸馏要求在同一批 128 个样本上比较 gold-SFT 与 KD，KD 相对 SFT 的 AUC 差值分别为 heart +0.7、blood +14.7、diabetes −0.5、car +1.1。9B 教师 OOT AUC / KS 为 0.72 / 0.45，0.8B 学生为 0.71 / 0.41。

相关工作沉淀为腾讯内网 KM 技术文章，并申请发明专利《一种基于大语言模型的自动化标签挖掘方法及系统》。对面试表达而言，这个项目的价值不是单一婚姻标签，而是一条可迁移到其他用户画像任务的标签开发链路。

Review｜边界、追问与待补证据

  • LLM / Agent 不适合全面替代 GBDT。大样本、高 QPS、以数值特征为主的任务仍优先使用树模型。
  • Agent 的优势是解释、冷启动与难例补充；稳态批量任务更适合微调后的小模型。
  • 蒸馏上限受教师约束，硬标签 AUC 与概率排序 AUC 必须明确口径。
  • 公开集实验用于说明方法，不等同于业务数据结果；业务表细节继续按脱敏要求处理。
  • 待补证据包括脱敏总架构图、KM 截图、专利可公开页、Lift / 消融图，以及个人与协作者分工说明。
